# OpenHAC — Open Hardware-as-Code

OpenHAC is a Python-based compiler that transforms declarative hardware code into fully manufacturable PCB designs. By bridging the gap between automated software pipelines and physical hardware manufacturing, OpenHAC allows developers to design, simulate, and generate production-ready boards entirely through code—no GUI required.

## What It Does

Write your hardware design in Python by defining components, wiring modules together, and setting spatial constraints. The OpenHAC compiler processes this logic and automatically produces:

- **Manufacturing Files:** Generated Gerber layers, Excellon drill files, and CPL (Component Placement List) files.
- **Automated BOMs:** Intelligent Bill of Materials strictly aligned with real-world supplier inventory and LCSC/JLCPCB-oriented fields.
- **Native CAD Output:** `.kicad_pcb` boards with outlines, footprint placement, pad nets, and rule-area keepouts.
- **Schematic & Project Stubs:** `.kicad_sch` and `.kicad_pro` files for manual review or finishing.
- **SPICE Simulations:** `.cir` netlists for ngspice-oriented circuit simulation.

## The Manufacturing Pipeline

OpenHAC is designed to create a frictionless pipeline from code generation directly to fabrication and SMT assembly. 

To achieve this, OpenHAC utilizes a live component database. Rather than relying on static, outdated libraries, the compiler dynamically queries real-time SMT component availability, technical specifications, and pricing. This ensures that the hardware configurations generated by the software are optimized for in-stock inventory and can be instantly routed to fabrication services for batch ordering.

## Example Syntax

Modules in OpenHAC group related components and expose named `Interface` connection points, abstracting away manual pin routing.

```python
from openhac.core import Board
from openhac.stdlib.mcu import ESP32_WROOM
from openhac.stdlib.power import XT60_Input, LDO_5V

# Initialize a 60x40mm 2-layer board
board = Board(size_mm=(60, 40), layers=2)

power = XT60_Input()
ldo   = LDO_5V()
mcu   = ESP32_WROOM()

board.add_module(power)
board.add_module(ldo)
board.add_module(mcu)

# Connect interfaces — no raw pin numbers needed at the board level
board.connect(power.v_out, ldo.v_in)
board.connect(ldo.v_out,   mcu.power)

# Apply automated spatial constraints for physical layout
board.constrain_edge(power, "TOP")
board.constrain_distance_min(ldo, mcu, 8) # Thermal isolation
